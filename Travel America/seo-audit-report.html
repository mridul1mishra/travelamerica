🔒 Security Issues

533 issues
275

HTTP (Non-HTTPS) URLs

275 URLs are served or linked over HTTP. The site appears to be running on localhost for this crawl, but the production domain should enforce HTTPS everywhere. HTTP URLs are flagged by Chrome as "Not Secure" and receive no HTTPS ranking boost.

258

Missing X-Frame-Options Header

258 pages lack the X-Frame-Options HTTP header, leaving them vulnerable to clickjacking attacks. Set X-Frame-Options: SAMEORIGIN (or use Content-Security-Policy: frame-ancestors) on all responses.

Particularly urgent for the booking tab pages (?tab=hotels, ?tab=flights, etc.) — add a canonical pointing to the base booking URL (without parameters). This will collapse 12 duplicate pages into their canonical counterparts.

    High  Fix duplicate and over-length page titles Give every page a unique title. The /destination/nyc/is-nyc-safe-at-night page is sharing the homepage title — that's a crawlability red flag. Trim any title over 60 characters. The booking tab pages need unique titles only if you don't canonicalize them (see #2).
  • 5
    High  Add X-Frame-Options security header Set X-Frame-Options: SAMEORIGIN in your Next.js next.config.js headers configuration for all routes. This is a one-line fix that closes 258 security warnings.
  • 6
    Medium  Boost internal links to near-orphaned pages /destination/la/neighborhood-guide and /destination/orlando/neighborhood-guide each have only 1 inlink. Add contextual links from the destination hub pages and relevant articles. Pages with low inlinks receive less PageRank and are crawled less frequently.

    Medium  Fix H1 and meta description duplicates Once canonical tags are in place (action #2), the booking tab pages will be non-indexable and their duplicate H1s/descriptions won't matter. For remaining duplicates, write unique H1s that accurately describe each page's primary topic.
