Resolved Since Last Review

The following items from the June 2 / June 14 reports were checked against the current code and are confirmed done. No further action needed.

All three redirects (/majorcities/lasvegas/solo-itinerary, /majorcities/orlando/solo-itinerary, /majorcities/losangeles/solo-itinerary) now correctly point to their respective /destination/<city>/solo-itinerary pages instead of NYC.

sitemap.ts now contains 64 routes covering all LA, Las Vegas, and Orlando content pages plus is-nyc-safe-at-night (was 22 routes / 39 missing). A small follow-up remains — see new finding below.

NYC, LA, Las Vegas, and Orlando hub pages (nycclient.tsx, lahubclient.tsx, lasvegashubclient.tsx, orlandohubclient.tsx) are now server components ("Server Component — no hooks used" comments confirm this). The most impactful hydration issue from the original report is resolved.

✓ High — error.tsx / not-found.tsx added

app/error.tsx, app/destination/error.tsx, app/not-found.tsx, and app/destination/not-found.tsx all exist now.

A shared app/components/booking/BookingClient.tsx (336 lines) now exists. The four "bookings" pages (labookingsclient.tsx, lasvegasbookingsclient.tsx, orlandobookingsclient.tsx, NYC's bookflightsclient.tsx) are now ~100-line config wrappers around it. However, see the new finding below — two old full duplicates were left behind as dead code.

✓ High — Schema markup gap

74 files now contain application/ld+json structured data, including LA/Vegas/Orlando pages (48 files) and NYC (18 files plus shared components). Only the two map-viewer "/full" pages lack schema, which is reasonable.

Tailwind has been fully removed (not in package.json, zero files use Tailwind utility classes). CSS Modules is now the sole styling system alongside plain CSS files (see remaining item below).
